# H1 — Add Hasura ops replacing all Sequelize queries (+ codegen)

**Repo:** sworld-backend
**Type:** foundation (data layer)
**Estimate:** M
**Blocked by:** None
**Blocks:** M1, M2, M3, M4, M5, M6
**Parallel-safe with:** — (owns all codegen; run alone)

## Context

Every call site currently imports from `src/database/queries/*` (Sequelize). To
let those sites migrate in parallel without fighting over the generated GraphQL
files, this ticket adds **all** the replacement Hasura operations in one place,
runs `pnpm codegen` once, and exposes typed helpers with the **same signatures**
as the Sequelize helpers they replace. After this lands, M1–M6 are pure import
swaps with no codegen.

Mirror the existing patterns in `src/services/hasura/queries/share/index.ts` and
`src/services/hasura/mutations/videos/finalize.ts` (combined mutation).

## Scope

New typed Hasura helpers (admin client, `graphql()` tagged documents, generated
types). Signatures must match today's Sequelize helpers so call sites swap 1:1.

**Videos queries** — `src/services/hasura/queries/videos/index.ts`:
- `getVideoById(id)` → `videos_by_pk(id)`; returns `{ id, source, status, user_id, duration, thumbnail_url }` (the fields the fix handlers read).
- `getVideoMissingDuration()` → `videos(where: { _or: [{duration: {_is_null: true}}, {duration: {_eq: 0}}] })`.
- `getVideoMissingThumbnail()` → `videos(where: { status: {_eq: "ready"}, _or: [{thumbnail_url: {_is_null: true}}, {thumbnail_url: {_eq: ""}}] })`.

**Videos combined mutations** (one document each, replacing a Sequelize transaction):
- `src/services/hasura/mutations/videos/fixDuration.ts` →
  `fixVideoDuration({ id, duration, taskId })`: `update_videos_by_pk(pk_columns:{id}, _set:{duration})` **and** `update_tasks(where:{task_id:{_eq:$taskId}}, _set:{status:"completed", completed:true})` in one mutation. Throw if either `affected_rows`/result indicates the video or task was not found (preserve the existing "not found" errors).
- `src/services/hasura/mutations/videos/fixThumbnail.ts` →
  `fixVideoThumbnail({ id, thumbnailUrl, taskId })`: same shape with `_set:{thumbnail_url}`.

**Tasks mutations** — `src/services/hasura/mutations/tasks/index.ts`:
- `createTask({ taskId, type, metadata, entityType, entityId })` → `insert_tasks_one(object:{...task_id...}, on_conflict:{constraint: tasks_task_id_key, update_columns: []}) { id completed status }`. Returns the row (existing on conflict) so callers can read `completed`. Keep the required-field guard that throws on missing inputs.
- `updateTaskStatus({ taskId, status })` → `update_tasks(where:{task_id:{_eq}}, _set:{status})`; throw if `affected_rows === 0` ("Task with ID … not found").
- `completeTask({ taskId })` → `update_tasks(where:{task_id:{_eq}}, _set:{status:"completed", completed:true})`; throw if `affected_rows === 0`.

**Codegen + tests:**
- Run `pnpm codegen` and commit the regenerated `src/services/hasura/generated-graphql/*`.
- Unit-test each helper the way existing Hasura helpers are tested (mock `hasuraClient.request`): correct document + variables, return mapping, and the throw-on-not-found / required-field paths.

## Acceptance criteria

- [ ] All helpers above exist, are typed from generated types, and use `hasuraClient`.
- [ ] `createTask` returns the existing row (with `completed`) on `task_id` conflict.
- [ ] `fixVideoDuration` / `fixVideoThumbnail` update video **and** task in a single mutation document.
- [ ] Not-found / missing-field error paths match the current Sequelize behaviour.
- [ ] `pnpm codegen` runs clean and the regenerated files are committed.
- [ ] Unit tests pass; `pnpm lint` and `tsc` are clean.
