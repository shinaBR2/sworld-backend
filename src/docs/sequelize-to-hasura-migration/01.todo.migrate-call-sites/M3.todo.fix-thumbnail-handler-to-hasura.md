# M3 — Migrate `fix-thumbnail` handler to Hasura (combined mutation)

**Repo:** sworld-backend
**Type:** migration (call site)
**Estimate:** S
**Blocked by:** H1
**Blocks:** D1
**Parallel-safe with:** M1, M2, M4, M5, M6

## Context

`fixThumbnailHandler` reads a video, parses its M3U8, generates a thumbnail, then
writes the thumbnail URL and completes the task inside a Sequelize transaction
(`updateVideoThumbnail` + `completeTask`). Swap the read to H1's `getVideoById`
and replace the two-write transaction with H1's single combined mutation
`fixVideoThumbnail`.

## Scope

- Replace `getVideoById` import with the H1 Hasura query
  (`src/services/hasura/queries/videos`).
- Remove the Sequelize transaction block (`sequelize.transaction()`,
  `updateVideoThumbnail` + `completeTask`, `commit`/`rollback`). Replace with a
  single `fixVideoThumbnail({ id, thumbnailUrl, taskId })` call
  (`src/services/hasura/mutations/videos/fixThumbnail`).
- Drop the `sequelize` import and the `let transaction` bookkeeping.
- Keep all existing behaviour: "Video not found", the empty-HLS / invalid-length
  guards, thumbnail generation, `shouldRetry` flags, the
  `VIDEO_ERRORS.FIX_THUMBNAIL_ERROR` wrapping, logging, and the
  `AppResponse(true, 'ok', { taskId })` return.
- Update `fix-thumbnail/index.test.ts` to mock the Hasura helpers; assert the
  not-found, invalid-length, success, and mutation-failure paths.
