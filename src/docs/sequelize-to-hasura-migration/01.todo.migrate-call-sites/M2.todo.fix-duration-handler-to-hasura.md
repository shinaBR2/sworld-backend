# M2 — Migrate `fix-duration` handler to Hasura (combined mutation)

**Repo:** sworld-backend
**Type:** migration (call site)
**Estimate:** S
**Blocked by:** H1
**Blocks:** D1
**Parallel-safe with:** M1, M3, M4, M5, M6

## Context

`fixDurationHandler` reads a video, parses its M3U8 for the duration, then writes
the duration and completes the task inside a Sequelize transaction
(`updateVideoDuration` + `completeTask`). Swap the read to H1's `getVideoById`
and replace the two-write transaction with H1's single combined mutation
`fixVideoDuration`.

## Scope

- Replace `getVideoById` import with the H1 Hasura query
  (`src/services/hasura/queries/videos`).
- Remove the Sequelize transaction block (`sequelize.transaction()`,
  `updateVideoDuration` + `completeTask`, `commit`/`rollback`). Replace with a
  single `fixVideoDuration({ id, duration, taskId })` call
  (`src/services/hasura/mutations/videos/fixDuration`).
- Drop the `sequelize` import and the `let transaction` bookkeeping.
- Keep all existing behaviour: the "Video not found" `CustomError`, the M3U8
  parse, the `VIDEO_ERRORS.FIX_DURATION_ERROR` wrapping, logging, and the
  `AppResponse(true, 'ok', { taskId })` return.
- Update `fix-duration/index.test.ts` to mock the Hasura helpers; assert the
  not-found path, the success path, and that a mutation failure still surfaces
  `FIX_DURATION_ERROR`.
