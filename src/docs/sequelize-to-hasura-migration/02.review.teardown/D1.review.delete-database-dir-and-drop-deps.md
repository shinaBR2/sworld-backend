# D1 — Delete `src/database/` + drop `sequelize` / `pg` / `pg-hstore`

**Repo:** sworld-backend
**Type:** teardown
**Estimate:** S
**Blocked by:** M1, M2, M3, M4, M5, M6, M7, M8, M9
**Blocks:** None
**Parallel-safe with:** — (must land last)

## Context

Once every consumer is migrated (M1–M6), the entire Sequelize layer is dead code.
Remove it and the now-unused dependencies. This is the ticket that actually
deletes the direct-to-Postgres path the epic set out to kill.

## Scope

- **Verify zero remaining usage first** (these must all return nothing):
  - `grep -rn "from 'sequelize'\|from \"sequelize\"" src`
  - `grep -rn "src/database" src`
  - `grep -rn "\bsequelize\b" src --include='*.ts' | grep -v docs`
- Delete the whole `src/database/` directory (`index.ts`, `models/`, `queries/`,
  and their `*.test.ts`).
- Remove `sequelize`, `pg`, and `pg-hstore` from `package.json` dependencies
  (confirm `pg` / `pg-hstore` have no other importer — they're Sequelize's
  Postgres driver/peer). Run the package manager to update the lockfile.
- Remove any now-orphaned references (e.g. a `databaseUrl` in `envConfig` /
  `*.env` examples) **only if nothing else reads it** — check first; leave it if
  anything else uses it.

## Files to touch (ownership)

- `src/database/**` (deleted)
- `package.json` (deps) + lockfile
- `src/utils/envConfig.ts` / env docs (only if `databaseUrl` is now unused)

## Acceptance criteria

- [ ] `src/database/` no longer exists.
- [ ] `sequelize`, `pg`, `pg-hstore` are gone from `package.json`; lockfile updated.
- [ ] No `sequelize` / `src/database` references remain anywhere in `src/` (outside this docs folder).
- [ ] `pnpm install`, `pnpm lint`, `pnpm build`, and the full `pnpm test` all pass.

## Test plan

- `pnpm install && pnpm lint && pnpm build && pnpm test:ci`

## Out of scope

- Any further Hasura work.
- Removing the legacy local CLI in `src/cli/` (separate concern).
