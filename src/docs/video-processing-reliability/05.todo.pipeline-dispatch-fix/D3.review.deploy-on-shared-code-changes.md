# D3 ŌĆö Deploy on shared-code changes (CI/CD path filters)

**Repo:** sworld-backend
**Type:** bug (CI/CD)
**Estimate:** XS
**Blocked by:** ŌĆö
**Blocks:** shipping D2 (and any future shared-code fix)

## The incident

Merging D2 (#161) to `main` triggered **no deploy** ŌĆö so the queue-ack fix sat in
`main` without ever running in PROD (the Cloud Tasks queue kept retrying).

## Root cause

Each of the 6 deploy workflows only fires on changes under its own
`src/apps/<service>/**` (+ that service's Dockerfile + `package.json`):

```yaml
paths:
  - 'src/apps/io/**'   # ŌåÉ misses shared code
```

But every service image bundles the **shared** code too ŌĆö `src/middleware/`,
`src/utils/`, `src/services/`, `src/schema/`, `src/types/`. D2 changed
`src/middleware/reportVideoFailure/**`, which matched no filter ŌåÆ no deploy. Any
shared-code change is silently never deployed. There was also no
`workflow_dispatch`, so no way to redeploy manually.

(D1 deployed only because it touched `src/apps/gateway/**`; its shared validator
change didn't redeploy io/compute.)

## The fix

The correct model: **each service deploys on its own code + shared code only** ŌĆö
never on a *different* service's app code. For all 6 deploy workflows
(dev + prod ├Ś io/compute/gateway), the push path filter is:

- **Own code:** `src/apps/<service>/**` + the entrypoint `src/<service>.ts`.
- **Shared code:** `src/middleware/**`, `src/schema/**`, `src/services/**`,
  `src/utils/**` (the dirs every service image bundles).
- Plus `Dockerfile.<service>`, `package.json`, `package-lock.json`.

So an io-only change redeploys **io alone**; a shared-code change redeploys all
three. (`src/cli/**`, `src/docs/**`, `src/test/**` are not deployed, so they
trigger nothing.)

Also add **`workflow_dispatch:`** so a service can be redeployed manually (needed
to ship an already-merged shared-code change like D2, and useful generally).

## Acceptance criteria

- [ ] A change under `src/middleware|schema|services|utils/**` triggers all three
      service deploys.
- [ ] A change under `src/apps/io/**` (or `src/io.ts`) triggers **only** io ŌĆö
      not gateway or compute.
- [ ] Each deploy workflow can be run manually from the Actions tab.

## Follow-up (manual, after merge)

- Trigger **prod-io** + **prod-compute** via `workflow_dispatch` to finally ship
  D2's queue-ack fix (already-merged history won't auto-trigger).

## Notes

- A shared-code change rebuilding all three services is correct (they all bundle
  it), not over-triggering ŌĆö that's the intended behaviour.
