# P3 — CLI `stream-m3u8` outputs fMP4

**Repo:** sworld-backend
**Type:** feature
**Estimate:** M
**Blocked by:** F1
**Blocks:** V1
**Parallel-safe with:** P1, P2

## Context

`src/cli/stream-m3u8.ts` is the local manual-processing tool. It is a
**standalone replica** — it imports **no** backend/core code and reimplements
everything inline (`slugify` "kept byte-for-byte", `resolveMasterPlaylist`,
`parseM3U8Content`, `uploadSegments`, a raw `FINALIZE_VIDEO_MUTATION`). So
fixing the backend (P2) does **not** fix the CLI — it needs its own change. It
has the same fragility as P2 (it produced the noisy Gosick episodes).

## Scope

- Add a **TS→fMP4 remux** before upload: emit `init.mp4` + `.m4s` and an fMP4
  playlist; upload all three with correct content-types/cache.
- **Prefer importing F1's `packageToFmp4` helper** instead of duplicating the
  ffmpeg logic — the CLI runs in-repo via `tsx`, so it can import from
  `src/services/...`. This kills the replica drift so the CLI never needs a
  parallel fix again. (If the import pulls in unwanted build weight, fall back to
  inlining the same ffmpeg flags.)
- Keep the CLI's existing behaviour: master→variant resolve, ad-strip,
  `--referer`, find-or-create playlist, `--standalone`, the `--video-id`
  finalize. Only the segment packaging changes.
- Note in the CLI help that output is now fMP4.
