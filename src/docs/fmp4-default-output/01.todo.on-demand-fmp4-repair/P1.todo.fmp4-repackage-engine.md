# P1 — fMP4 repackage engine (stored `.ts` → fMP4)

**Repo:** sworld-backend
**Type:** feature
**Estimate:** S–M
**Blocked by:** F1 (done — reuses the core's ports/DI pattern)
**Blocks:** P2, V1
**Parallel-safe with:** —

## Context

The repair needs a reusable engine that takes the `.ts` we **already stored** for
one video and repackages it into fMP4/CMAF (`init.mp4` + `.m4s`), **without
re-encoding the video**. This is the engine only — no CLI wiring, no GCS swap
logic (that's P2). It is **purely additive**: it does not touch `processStream`,
the stream-hls handler, or the convert path. The default streaming flow keeps
emitting `.ts` exactly as today.

This is the codified version of the manual procedure that fixed Gosick during the
incident (read the stored `.ts` playlist → ffmpeg remux → fMP4).

## Scope

- Add `repackageToFmp4(input, deps)` under `src/services/videos/processing/`
  (sibling to `processStream`), following the same injectable-deps pattern (F1):
  - **input**: a reference to the stored `.ts` playlist for one video (the GCS
    path / a readable playlist + its segments).
  - **deps**: `storage` (read stored objects, write new ones), an `ffmpeg` port,
    `logger`. No http/hasura — repair works off bytes we own.
- Run the README ffmpeg recipe: `-c:v copy` (no video transcode) + **audio
  re-encode** `-c:a aac -b:a 128k -ac 2 -ar 44100`, `-hls_segment_type fmp4`,
  `-hls_fmp4_init_filename init.mp4`, `%d.m4s`. Re-encode is the proven-clean
  choice (regenerates a clean `AudioSpecificConfig`); `aac_adtstoasc` copy is the
  cheaper alternative but reuses the malformed source config — not the default.
- Output: `init.mp4` (`content-type: video/mp4`) + `.m4s` segments + a playlist
  with `#EXT-X-MAP` + `#EXT-X-VERSION:7`, whatever ffmpeg writes.
- Return what was produced (new object names, playlist) so P2 can do the swap.

## Files to touch (ownership)

- `src/services/videos/processing/` — **P1 owns the new `repackageToFmp4` engine
  + its ffmpeg port.** Does NOT modify `processStream` or existing handlers.

## Acceptance criteria

- [ ] `repackageToFmp4` turns a stored `.ts` playlist into `init.mp4` + `.m4s` +
      an `#EXT-X-MAP` playlist.
- [ ] `-c:v copy` (no video re-encode); `ffprobe -map 0:a -f null -` on the
      output decodes clean.
- [ ] Engine is env-free and unit-testable with fake deps.
- [ ] No change to `processStream` / stream-hls / convert behaviour.

## Test plan

- Unit: feed a small fixture playlist → assert fMP4 output shape (init + `.m4s` +
  `EXT-X-MAP`), audio decode clean.

## Out of scope

- CLI command, GCS read/swap/cleanup, DB lookup (all P2).
- Touching the default streaming or convert flows (explicitly not in this epic).
- Batch/catalog repair (G1).
