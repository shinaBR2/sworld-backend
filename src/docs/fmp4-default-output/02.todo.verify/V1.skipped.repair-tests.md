## Status: SKIPPED

- The **CLI** (P2) is coverage-excluded operator tooling, and it was verified
  **end-to-end on PROD**: a real noisy Gosick segment streamed to `.ts`, repaired

## Context

Lock in the repair's two guarantees: it produces valid fMP4, and it never leaves
a broken video. Note what we are **not** testing: the default streaming flow
still emits `.ts` **on purpose**, so there is **no** "ban `.ts`" assertion here.

## Scope

- **Format (P1)**: repackage output is fMP4 — playlist has
  `#EXT-X-MAP:URI="init.mp4"` + `.m4s` (not `.ts`); `init.mp4` is `video/mp4`;
  `ffprobe -map 0:a -f null -` on the output decodes clean.
- **Safety (P2)**: with fake GCS, assert the order **upload new → swap playlist →
  verify → delete old**, and that a thrown repackage/upload aborts with the
  original `.ts` left fully intact (nothing deleted, playlist unchanged).
- One small end-to-end on a known fragile fixture → repaired output is fMP4.

## Files to touch (ownership)

  assertions.**

## Acceptance criteria

- [ ] A test fails if the repair emits `.ts` instead of fMP4.
- [ ] A test fails if delete can happen before the new playlist verifies.
- [ ] A test proves a failed repair leaves the original video intact.
- [ ] CI green.

## Out of scope

- Asserting anything about the default streaming/convert output (unchanged).
- Perceptual/browser audio testing (covered manually per P2).
- Batch repair (G1).
