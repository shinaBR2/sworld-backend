## Context

Lock the format in so we don't silently regress to `.ts`.

## Scope

- Unit tests asserting each path's output is fMP4:
  - playlist contains `#EXT-X-MAP:URI="init.mp4"` and `.m4s` (not `.ts`).
  - `init.mp4` is uploaded with `content-type: video/mp4`.
  - `ffprobe -map 0:a -f null -` on the output is **clean** (no decode errors).
- One small e2e (or integration) that processes a known HE-AAC fixture end to
  end and asserts fMP4 output.

## Files to touch (ownership)

- test files alongside F1/P1/P2/P3 — **V1 owns the format assertions.**

## Acceptance criteria

- [ ] A test fails if any path emits `.ts` again.
- [ ] CI green.

## Out of scope

- Browser/audio-perceptual testing (covered manually per P1–P3).
- Catalog migration (G1).
