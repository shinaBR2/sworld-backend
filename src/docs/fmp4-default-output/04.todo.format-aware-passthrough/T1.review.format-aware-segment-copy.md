# T1 — Format-aware segment passthrough (.ts stays .ts, .m4s stays .m4s)

**Repo:** sworld-backend
**Type:** fix / feature
**Estimate:** S
**Blocked by:** F1 (the shared core)
**Blocks:** —

## Context

The stream path (backend `stream-hls` + CLI `stream`, both via `processStream`)
is a **byte-copy proxy**: it copies source segments unchanged and rewrites the
playlist. But it **hardcoded** `.ts` naming + `video/MP2T` content-type and
ignored `#EXT-X-MAP`. So:

- A `.ts` source → fine (it is `.ts`).
- An **fMP4 source** (`.m4s` + init) → silently broken: segments renamed `.ts`,
  mislabeled `video/MP2T`, and the **init segment dropped** → unplayable.

We've only avoided this because every source so far is `.ts`. This makes the
proxy faithfully pass through whatever the source already is — **no ffmpeg**.
(Converting `.ts` → fMP4 remains the on-demand `repair-fmp4` tool's job.)

## Scope

- `parseHlsManifest`: detect format from `#EXT-X-MAP` — fMP4 → keep `.m4s` and
  carry the init (renamed `init.mp4`, re-emit `#EXT-X-MAP:URI="init.mp4"`); else
  `.ts` (unchanged). Return the init in the parsed result.
- `processStream`: content-type is now per-extension (`.ts` → `video/MP2T`,
  `.m4s` → `video/iso.segment`, `init.mp4` → `video/mp4`); fetch + upload the init
  (once) when present.
- **Byte-range** sources (single-file fMP4/TS) are unsupported by a whole-file
  copy → **fail loud** (terminal `INVALID_VIDEO_FORMAT`), not a broken video.

## Acceptance criteria

- [x] `.ts` source → output unchanged (`.ts`, `video/MP2T`, no `#EXT-X-MAP`).
- [x] `.m4s` source → output keeps `.m4s` + `init.mp4` + `#EXT-X-MAP`, correct
      content-types.
- [x] Byte-range source → terminal error.
- [x] Shared core, so backend **and** CLI both get it. Full suite green.

## Known limitation (acceptable)

Thumbnail generation for an fMP4 source is best-effort and will likely no-op (the
per-segment ffmpeg uses `mpegts` input and an `.m4s` needs its init) — it's
caught and skipped, never failing the video. No fMP4 sources exist today; revisit
if/when they do.
