# F1 — fMP4 packaging helper + playlist/init upload

**Repo:** sworld-backend
**Type:** foundation
**Estimate:** S
**Blocked by:** —
**Blocks:** P1, P2, P3
**Parallel-safe with:** —

## Context

All three output paths (convert, stream-hls, CLI) currently emit MPEG-TS `.ts`
segments + a media playlist that lists them. To switch to fMP4 we need one
shared place that knows the fMP4 shape, so the format decision lives in exactly
one spot and the three paths just call it.

## Scope

- A small **`packageToFmp4`** helper (one module) that, given a local source
  (a concatenated TS / a media-playlist of segments / an mp4), runs the ffmpeg
  recipe in the epic README and yields `{ initPath, segmentPaths[], playlist }`.
  - `-c:v copy` always; audio `-c:a aac -b:a 128k -ac 2 -ar 44100` (or
    `-c copy -bsf:a aac_adtstoasc` if we confirm pure-remux output plays clean).
  - `hls_segment_type fmp4`, `hls_fmp4_init_filename init.mp4`, `%d.m4s`.
- **Upload** support for the **init segment**: `init.mp4` with
  `content-type: video/mp4`, long cache; `.m4s` with `content-type: video/mp4`,
  long cache; `playlist.m3u8` with `content-type: application/x-mpegURL`,
  `no-cache`.
- **Playlist** is whatever ffmpeg writes (`#EXT-X-VERSION:7`,
  `#EXT-X-MAP:URI="init.mp4"`, `.m4s`) — don't hand-roll it.

## Files to touch (ownership)

- `src/services/videos/helpers/fmp4/` (new) — the packaging helper.
- the storage/upload util used by processing (content-type + cache for
  `init.mp4` / `.m4s`). **F1 owns the fMP4 upload content-type map.**

## Acceptance criteria

- [ ] Helper turns a source into `init.mp4` + `.m4s` + a valid fMP4 playlist.
- [ ] `ffprobe` on the output decodes audio with **no errors**; playlist has
      `#EXT-X-MAP`.
- [ ] Uploaded objects carry the correct content-types and cache-control.
- [ ] No video transcode (verify with `-c:v copy`; encode time ≈ remux time).

## Test plan

- Unit: run the helper on a small TS fixture; assert playlist has `EXT-X-MAP`,
  init + `.m4s` exist, `ffprobe -map 0:a -f null` is clean.

## Out of scope

- Wiring any of the three paths to call it (P1–P3).
- Migrating existing catalog content (G1).
